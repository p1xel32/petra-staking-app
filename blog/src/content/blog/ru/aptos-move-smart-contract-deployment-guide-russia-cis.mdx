## Введение в Aptos и язык Move

Прежде чем перейти к практическим шагам, важно понять фундаментальные технологии, лежащие в основе Aptos.

### Aptos: Блокчейн нового поколения для Web3

**Aptos** — это Layer 1 блокчейн, созданный с нуля с акцентом на безопасность, масштабируемость и пользовательский опыт. Его уникальные особенности включают:
*   **Параллельное исполнение транзакций (Block-STM)**: Позволяет обрабатывать тысячи транзакций в секунду, значительно повышая пропускную способность.
*   **Модульность и обновляемость**: Aptos разработан таким образом, чтобы быть легко обновляемым, что позволяет быстро внедрять новые функции и улучшения.
*   **Безопасность**: Использует язык программирования Move, разработанный специально для безопасного управления цифровыми активами.
*   **Низкие комиссии и быстрая финализация**: Делает сеть привлекательной для широкого спектра децентрализованных приложений (dApps).

Цель Aptos — стать надежной и высокопроизводительной основой для массового внедрения Web3-технологий по всему миру, включая регион СНГ.

### Move: Язык смарт-контрактов, ориентированный на безопасность

**Move** — это инновационный язык программирования для смарт-контрактов, который изначально был разработан командой Meta (ранее Facebook) для проекта Diem. Он был создан с учетом присущей блокчейну потребности в безопасном и надежном управлении цифровыми активами. Ключевые преимущества Move включают:

*   **Типы ресурсов (Resources)**: В Move активы представлены как "ресурсы", которые не могут быть скопированы или неявно удалены. Это предотвращает распространенные проблемы, такие как двойная трата или потеря активов.
*   **Модульность**: Код организован в модули, что улучшает читаемость, переиспользуемость и безопасность за счет инкапсуляции.
*   **Формальная верификация**: Move разработан для облегчения формальной верификации, позволяя математически доказывать корректность смарт-контрактов, что критически важно для высокоценных приложений.
*   **Гибкость**: Позволяет разработчикам создавать сложные и выразительные смарт-контракты с акцентом на безопасность.

Именно благодаря Move **Aptos Move смарт-контракты** являются одними из самых надежных и безопасных в индустрии.

## Подготовка к развертыванию: Ваша среда разработки

Для того чтобы начать разработку и **развертывание смарт-контрактов Move на Aptos**, вам потребуется настроить локальную среду.

### 1. Установка Aptos CLI

**Aptos CLI** (Command Line Interface) — это основной инструмент для взаимодействия с сетью Aptos. Он позволяет управлять аккаунтами, развертывать контракты, вызывать функции и многое другое.

**Для Linux и macOS:**

Откройте терминал и выполните следующую команду:

```bash
curl -fsSL https://aptos.dev/aptos-cli/install.sh | bash
```

Следуйте инструкциям на экране. После установки убедитесь, что `aptos` добавлен в ваш PATH.

**Для Windows:**

Рекомендуется использовать [WSL (Windows Subsystem for Linux)](https://learn.microsoft.com/ru-ru/windows/wsl/install) или официальный установщик `aptos-cli.exe` из [репозитория GitHub](https://github.com/aptos-labs/aptos-core/releases).

**Проверка установки:**

После установки выполните:

```bash
aptos --version
```

Вы должны увидеть номер версии Aptos CLI.

### 2. Настройка кошелька Aptos

Для развертывания контрактов и оплаты транзакционных комиссий вам понадобится аккаунт Aptos с токенами.
*   **Создание аккаунта**: Вы можете использовать CLI (`aptos init`) или популярные расширения для браузера, такие как **Petra Wallet** или **Pontem Wallet**. Для начинающих рекомендуется использовать браузерный кошелек, так как он упрощает управление активами и подпись транзакций.
*   **Получение тестовых токенов**: Для работы в тестовых сетях (Devnet, Testnet) вы можете получить бесплатные тестовые токены через функции "Faucet" в вашем кошельке или через CLI:
    ```bash
    aptos account fund-with-faucet --account <ваш_адрес_кошелька> --url https://fullnode.devnet.aptoslabs.com --faucet-url https://faucet.devnet.aptoslabs.com
    ```
    Замените `<ваш_адрес_кошелька>` на реальный адрес вашего Aptos кошелька.

### 3. Создание нового проекта Move

Используйте Aptos CLI для инициализации нового проекта Move:

```bash
aptos move init <имя_вашего_проекта>
```

Например:

```bash
aptos move init my_first_aptos_contract
```

Эта команда создаст директорию `my_first_aptos_contract` со следующей структурой:
*   `sources/`: Здесь будут находиться ваши файлы `.move` со смарт-контрактами.
*   `Move.toml`: Файл конфигурации проекта Move, где указываются зависимости и адреса.

Откройте `Move.toml` и в секции `[addresses]` укажите адрес вашего аккаунта (полученный от Petra Wallet или Aptos CLI) для **именованного адреса** вашего модуля. Например:

```toml
[addresses]
my_first_aptos_contract = "0x..." # Замените на ваш 0x-адрес
```

Этот именованный адрес будет использоваться внутри ваших Move-модулей для ссылки на ваш собственный код.

## Разработка смарт-контракта на Move

Теперь, когда ваша среда готова, давайте напишем простой **Aptos Move смарт-контракт**.

### Пример простого контракта "Hello, Aptos!"

Создайте файл `sources/my_first_aptos_contract.move` в вашем проекте и добавьте следующий код:

```move
/// sources/my_first_aptos_contract.move
module my_first_aptos_contract::my_first_aptos_contract {
    use std::signer;

    /// Структура для хранения сообщения (опционально, для более сложных контрактов)

        value: vector<u8>,

    }

        // который вызвал эту функцию, демонстрируя, как работать с адресами.
        let sender_address = signer::address_of(account);

        // или использовать его для проверки разрешений.

        b"Hello, Aptos from Move!"
    }

    /// Пример функции для обновления состояния (если бы StoredMessage использовался)
    public entry fun update_message(account: &signer, new_message: vector<u8>) acquires StoredMessage {

        // let message_ref = borrow_global_mut<StoredMessage>(signer::address_of(account));
        // message_ref.value = new_message;
        // Для этого примера просто демонстрируем сигнатуру.
    }
}
```

**Объяснение кода:**
*   `module my_first_aptos_contract::my_first_aptos_contract`: Определяет модуль. `my_first_aptos_contract` слева от `::` — это именованный адрес, который вы указали в `Move.toml`. Правая часть — это имя самого модуля.
*   `use std::signer;`: Импортирует стандартную библиотеку `signer` для работы с подписывающими аккаунтами.
*   `struct StoredMessage`: Пример структуры данных. Ключевое слово `has key, store, drop` определяет свойства структуры: `key` позволяет ей быть глобальным ресурсом, `store` позволяет хранить ее внутри других структур, `drop` означает, что ее можно безопасно удалить.
*   `public entry fun hello_aptos(account: &signer): vector<u8>`:
    *   `public`: Функция видна извне модуля.
    *   `entry`: Эта функция может быть вызвана как отдельная транзакция извне блокчейна (через CLI или dApp).
    *   `account: &signer`: Аргумент `signer` представляет аккаунт, который инициирует транзакцию, и используется для проверки подписи и доступа к глобальному состоянию.
    *   `vector<u8>`: Тип возвращаемого значения (байтовый массив, который может представлять строку).
*   `b"Hello, Aptos from Move!"`: Это байтовая литерал-строка.

### Компиляция контракта Move

После написания кода вам необходимо его скомпилировать. Это проверит синтаксис, типобезопасность и сгенерирует байткод, который будет развернут в блокчейне.

Перейдите в корневую директорию вашего проекта (`my_first_aptos_contract`) в терминале и выполните:

```bash
aptos move compile --named-addresses my_first_aptos_contract=<ваш_0x_адрес>
```

Замените `<ваш_0x_адрес>` на адрес, который вы указали в `Move.toml`.
Если компиляция прошла успешно, вы увидите сообщение о том, что байткод сгенерирован в `build/my_first_aptos_contract/bytecode_modules/my_first_aptos_contract.mv`.

## Развертывание смарт-контракта в сети Aptos

Теперь, когда контракт скомпилирован, мы готовы к его **развертыванию Aptos Move смарт-контракта**.

### Выбор сети Aptos

Aptos предлагает несколько сред для развертывания:
*   **Devnet**: Разработческая сеть, часто обновляется, идеально подходит для раннего тестирования.
*   **Testnet**: Более стабильная тестовая сеть, ближе к Mainnet, используется для более серьезного тестирования и интеграции.
*   **Mainnet**: Основная сеть, где работают реальные активы. Развертывание здесь требует максимальной осторожности и реальных токенов APT.

Для этого руководства мы будем использовать **Devnet**.

### Непосредственное развертывание

Выполните следующую команду для публикации вашего модуля:

```bash
aptos move publish \
    --named-addresses my_first_aptos_contract=<ваш_0x_адрес> \
    --private-key <ваш_приватный_ключ> \ # ОСТОРОЖНО: В продакшене используйте KMS или Wallet
    --url https://fullnode.devnet.aptoslabs.com
```

**Важные примечания:**
*   Замените `<ваш_0x_адрес>` на ваш реальный 0x-адрес.
*   Замените `<ваш_приватный_ключ>` на приватный ключ вашего аккаунта. **Будьте крайне осторожны с приватными ключами! Никогда не используйте этот метод для аккаунтов с реальными средствами на Mainnet.** В реальной разработке используйте безопасные методы подписи транзакций через кошельки (Petra Wallet, Pontem Wallet) или системы управления ключами.
*   `--url https://fullnode.devnet.aptoslabs.com`: Указывает на Devnet. Для Testnet используйте `https://fullnode.testnet.aptoslabs.com`.

Если вы не хотите использовать `--private-key` напрямую, CLI может запросить подтверждение через локально инициализированный кошелек или подключиться к вашей Ledger. Для простоты и безопасности в этом руководстве, особенно для тестовых сетей, использование ключа может быть приемлемо, но помните о рисках.

После успешной публикации вы увидите хеш транзакции. Ваш смарт-контракт теперь развернут в блокчейне Aptos!

### Взаимодействие с развернутым контрактом

Давайте вызовем функцию `hello_aptos` нашего развернутого контракта.

```bash
aptos move run \
    --function-id <ваш_0x_адрес>::my_first_aptos_contract::hello_aptos \
    --url https://fullnode.devnet.aptoslabs.com
```

Замените `<ваш_0x_адрес>` на адрес, под которым вы развернули контракт.

Вы должны увидеть вывод, похожий на:

```json
[
  "Hello, Aptos from Move!"
]
```

Это означает, что вы успешно вызвали функцию вашего **Aptos Move смарт-контракта** в сети!

## Безопасность и лучшие практики для разработчиков из СНГ

Поскольку Aptos набирает популярность в России и СНГ, крайне важно придерживаться лучших практик безопасности:

*   **Аудит кода**: Всегда проводите тщательный аудит вашего смарт-контракта, желательно с помощью сторонних экспертов.
*   **Тестирование**: Пишите всесторонние юнит- и интеграционные тесты для покрытия всех возможных сценариев использования.
*   **Move Prover**: Используйте встроенный в Move инструмент формальной верификации (Move Prover), чтобы математически доказать корректность критически важных свойств вашего контракта.
*   **Управление ключами**: Никогда не храните приватные ключи в открытом виде. Используйте аппаратные кошельки, системы управления ключами (KMS) или секретные менеджеры.
*   **Итеративная разработка**: Начинайте с малого, тестируйте в Devnet, затем в Testnet, и только потом развертывайте в Mainnet.
